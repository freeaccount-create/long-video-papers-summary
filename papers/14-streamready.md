## 2. 数据 / 输入格式（ProReady-QA）

- **规模**：10 段 1h Ego-4D + 22 段半小时 MovieNet，约 **5k 条主动式 QA**，每条带标注证据窗 `[t_s, t_e]`（§4.2）。源视频来自 VStream-QA，但删除全部"过去依赖"问题，改造为"未来依赖"的多轮主动问题。
- **五类任务**（§4.1）：SSR 步骤识别 / REC 重复事件计数 / CRR 线索揭示作答 / **CTD 因果触发检测** / GSD 目标完成检测。
- **关键标注**：每个 QA 标注首个有效证据帧 `t_s` 与末个有效证据帧 `t_e`，支持"问题先于证据出现"的主动场景。
- **输入流组织**：帧按时间顺序流式进入；问题在某时刻插入，模型以提示 `"Answer whenever you are ready"` 持续观察后续帧。
- **代表性样例**（**自构造示意，非论文原文样例**；论文未给出具体 CTD 文本样例，Fig.1/Fig.10 仅作泛化/对比示意）：
  ```
  视频: 1h 行车记录流; 问题在 t=0s 提出
  Q(CTD): "那辆红色的车撞上了吗?"
  evidence window: [t_s=20s, t_e=26s]   # 碰撞证据 20s 才出现, 26s 后画面切走
  GT answer: "撞上了"
  ```
  （此例仅为帮助理解"证据滞后于提问"的 CTD 任务而虚构，数值/场景非论文标注条目。）

## 3. 完整方法流程（含公式）

整体（§3.1）：流式帧 → 分层记忆 → 问题到达后 query-aware 检索推理 → `<RDY>` 就绪门控决定是否作答。

**(a) 记忆存储（§3.2）**：双记忆。
- 视觉记忆树 `M_V`：`M_V1` 最近帧 FIFO；满后 K-means 压成中层质心 `M_V2`，逐出帧以 EMA 更新（Eq.1）：
  `c_j ← (1-α)c_j + α f_o, if sim(f_o,c_j)≥τ_t; 否则新建质心`；质心再抽象成顶层原型 `M_V3`（Eq.2）。
- 上下文记忆库 `M_C`：存历史 `(q_i, a_i)`，供多轮复用。

**(b) Query-Aware 推理（§3.3）**：双分支 Q-Former。
- 短期：`z_s = Q_s(Concat[M_V1, q_i])`（Eq.3）。
- 长期：先在原型上 Top-K `M_V3^t = Top-K((W_s q_i)^T M_V3)`（Eq.4，softmax 归一化），再在其质心上 Top-m `M_V2^t = Top-m((W_c q_i)^T c)`（Eq.5，**不归一化**以保留尖锐排序），最后 `z_ℓ = Q_ℓ(Concat[M_V3^t, M_V2^t, q_i], z_s)`（Eq.6）。
- 上下文推理：`M_C` 软门控匹配历史 QA，cross-attention 融入 `z_ℓ`。

**(c) Readiness 机制（§3.4，核心）**：
- 把可学习 `<RDY>` token **挂在长期表征 `z_ℓ`（在 `Q_ℓ(·)` 内部）**，使其与"驱动答案生成的同一证据表征"共同演化。消融证实三种位置中挂在 `z_ℓ` 上最佳，挂短期分支噪声大、作为长期分支输入则中等但不稳定。
- 轻量 **Readiness Head（MLP，与 Transformer head 同效但更省）** 监控该 token，**每步输出** `R_pred ∈ [0,1]`。
- **阈值门控**：`R_pred > 阈值(0.35)` 才触发 LLM 作答，否则继续观察后续帧；就绪时把 `z_ℓ` 与 `M_C` 融合送 LLM，并把融合表征作为 `a_i` 存回 `M_C`。

**(d) 训练损失（弱监督，无需 GT 时间戳）**：用 `z_ℓ` 与质心记忆 `M_V2` 相似度构造伪正区 `P` / 伪负区 `N`，成对对比损失（Eq.7）：
`L_ctr = -log σ(R_pred(t⁺) - R_pred(t⁻)), t⁺∈P, t⁻∈N`；加时间平滑正则（Eq.8）：`L_rdy = L_ctr + λ_reg ‖∇_t R_pred(t)‖_1`。
**梯度只更新 Readiness Head 与 `<RDY>` token，对推理模块 stop-gradient**——"what"与"when"解耦学习。

**(e) 非对称早/晚惩罚（评测指标 ARS，§4.3）**：`ARS = (1/N)Σ EP_i·LP_i; Acc_e = Acc × ARS`（Eq.9）
- 早答惩罚（重，Eq.10）：`EP = softmin(1, 2·σ(γ_e·(t_a - t_s)/(τ+ε)))` —— `t_a→t_s` 时 EP→1；早答急剧衰减（乘 2 保证 `t_a=t_s` 恰满分）。
- 晚答惩罚（轻，Eq.11）：`LP = softmin(1, softmax(0, 1 - γ_ℓ·(t_a - t_e)/(τ+ε)))` —— 轻微延迟近 1，长延迟才下降。
- 取 `γ_e=6 ≫ γ_ℓ=1`，实现**早答惩罚 > 晚答惩罚**的非对称性；`τ`=证据时长中位数做尺度归一。

## 4. 方法流程走查（**自构造示意例**，非论文样例）

> 注：论文未提供红车/碰撞这一具体样例，本节用一个虚构 CTD 场景**演示方法流程**（机制步骤忠实于论文 §3.4 与 Eq.9-11，但 `t_s=20s/t_e=26s` 等数值为示意）。参数：`γ_e=6, γ_ℓ=1, 阈值=0.35`（论文取值）。

1. **t=0s 提问**"那辆红车撞上了吗？"。记忆树无碰撞证据，`z_ℓ` 与问题弱对齐、低置信，`<RDY>` 处于"扩散态"。
2. **t=0→19s 边收帧**：每步算 `R_pred`。证据未到、retrieval 与问题不一致，`R_pred` 保持低位（<0.35）→ **沉默继续观察**（若强行作答：`t_a<t_s`，EP 因 `γ_e=6` 迅速趋 0，是最重惩罚——避免幻觉抢答）。
3. **t≈20s 证据窗到达**：碰撞帧进入 `M_V1`→K-means 进 `M_V2`；Eq.4/5 检索命中该质心，`z_ℓ` 转向"含答案态"，`<RDY>` 表征跃升，`R_pred` 越过 0.35。
4. **触发作答**：`t_a≈20–26s ∈ [t_s,t_e]` → `EP≈1`、`LP≈1`，ARS≈1；答案"撞上了"正确 → `Acc_e=Acc×ARS` 接近满分。融合表征写回 `M_C` 供后续轮复用。
5. **对照**：基线（如 StreamBridge）就绪与推理解耦，常在证据窗后才触发→晚答 LP 惩罚，或问题时刻幻觉抢答→早答 EP 惩罚。

## 5. 模型 / 组件

- **Base streaming VideoLLM 主干**：Qwen-2-VL **7B**；另验证多种 7B MLLM 主干鲁棒（Table 8）。
- **双分支 Q-Former**（短期 `Q_s` / 长期 `Q_ℓ`），由 HierarQ 预训练权重初始化。
- **分层视觉记忆树**：FIFO 原始帧 / K-means+EMA 质心 / 原型；最佳配置 **24 帧 + 96 质心 + 12 原型**；检索取 8 原型 + 24 质心。
- **上下文记忆库 `M_C`** + soft-gating cross-attention。
- **Readiness 机制**：1 个 `<RDY>` token + **轻量 MLP Readiness Head**（无额外推理开销）。
- **评测指标**：Acc、ARS、Acc_e；对比基线 PAUC。

## 6. 创新点

1. **就绪感知形式化 + ARS 指标**：首次把"何时作答"与"答得对不对"联合建模，提出带**非对称早/晚惩罚**（`γ_e=6` 重罚抢答、`γ_ℓ=1` 轻罚延迟）的 ARS 与有效准确率 `Acc_e=Acc×ARS`（Eq.9-11）。
2. **证据耦合的轻量就绪机制**：`<RDY>` token 内嵌长期推理表征 `z_ℓ`、与驱动答案的同一证据共演化；伪监督对比损失 `L_ctr`（Eq.7-8）且对推理模块 stop-gradient，使 when 与 what 解耦——相比 StreamBridge 显著提升 ARS。
3. **ProReady-QA 基准**：首个带**标注答案证据窗 `[t_s,t_e]`** 的主动式多轮长流视频基准（含 CTD/GSD 等高阶因果/目标任务），支持系统化的时机评测。
